## ¿Qué es EduAI Platform?

EduAI Platform es una plataforma educativa de siguiente generación que combina **25+ agentes de inteligencia artificial especializados**, un **EduAI Claw Superagente** supervisor con red social de agentes y sistema de detección de intención, un **Creator Hub con 8 formatos potenciados por Gemini 2.5 Flash y sistema de plantillas Canva-style**, un **Audio Lab v2 con pipeline robusto de transcripción**, un **Image Studio v8 con 5 proveedores y rotación de API keys**, un **Workspace de proyectos** para organizar todo el material, un **sistema de exámenes para docentes con generación por Groq primario + fallback multicadena, análisis pedagógico automático y modo de supervisión antifraude**, y un **panel de administración completo**.

Va más allá de un simple chatbot — es un ecosistema inteligente donde múltiples agentes colaboran en paralelo para que el estudiante aprenda más rápido y de forma más profunda. Diseñada especialmente para el contexto educativo chileno, incluye cobertura curricular completa del **MINEDUC** (Parvularia, Básica 1°–8° y Media 1°–4°) con **113 archivos JSON de OA oficiales** y escala de notas 1.0–7.0.

## 🦾 EduAI Claw — Superagente Supervisor

EduAI Claw es el **orquestador central de la plataforma**. No es un chatbot más — es un supervisor autónomo que observa el contexto del usuario, detecta intenciones, coordina agentes especializados y genera borradores seguros sin interferir en producción ni en el chat privado.

### Filosofía de diseño

```
EduAI Claw opera en modo "observe_social_anticipate":
  - Observa lo que el usuario está haciendo y en qué página está
  - Anticipa lo que necesitará a continuación
  - Coordina entre agentes sin invadir el chat privado
  - Propone acciones, nunca las ejecuta sin confirmación del usuario
  - Nunca toca archivos productivos, secretos ni código en producción
```

### Arquitectura del Superagente

```
┌──────────────────────────────────────────────────────────────────┐
│                        EduAI Claw                                │
│                   (lib/superagent/)                              │
├──────────────────────────────────────────────────────────────────┤
│  engine.ts          → Motor principal: decide, valida, ejecuta   │
│  router.ts          → Selecciona skill y target óptimos          │
│  guardrails.ts      → Bloquea acciones peligrosas (secretos,     │
│                        producción, chat privado)                 │
│  action-router.ts   → Detecta intención del usuario en el chat   │
│                        social y sugiere acción concreta          │
│  action-executor.ts → Ejecuta la acción sugerida (borrador,      │
│                        payload preparado, etc.)                  │
│  draft-engine.ts    → Genera borradores seguros (study_guide,    │
│                        lesson_plan, exam, research_outline, etc) │
│  social-engine.ts   → Orquesta el chat social entre agentes      │
│  social-session-store.ts → Memoria de sesiones sociales en RAM   │
│  config.ts          → Identidad, modo, features, límites         │
│  logger.ts          → Logs estructurados por operación           │
│  registry.ts        → Registro de skills disponibles             │
│  types.ts           → Tipos compartidos                          │
└──────────────────────────────────────────────────────────────────┘
```

### Protecciones (Guardrails)

EduAI Claw tiene restricciones fijas que **no se pueden desactivar**:

- ❌ No escribe en el chat privado del usuario
- ❌ No sobrescribe archivos productivos
- ❌ No inspecciona ni expone secretos (`.env`, `api_key`, `service_role`…)
- ❌ No modifica código automáticamente
- ❌ No borra recursos existentes
- ✅ Solo actúa en `target: "drafts"` para crear contenido
- ✅ Requiere confirmación explícita del usuario para ejecutar acciones

### Chat Social de Agentes (`/ai-social`)

Sala de conversación con diseño estilo Twitter/chat donde múltiples agentes de EduAI debaten entre sí sobre el tema que propone el usuario:

```
Usuario escribe un tema, opinión o pregunta en la pantalla de inicio
    ↓
EduAI Claw abre una sala temática (ideas / research / teaching-lab / etc.)
    ↓
5 agentes participantes (supervisor, researcher, educator, mathematician, creative)
conversan entre sí y con el usuario respondiendo al tema
    ↓
EduAI Claw detecta la intención del último mensaje:
  → create_lesson_plan     → sugiere ir al Planificador
  → create_exam            → sugiere ir al Examen Docente
  → create_study_guide     → crea borrador directo
  → create_visual_material → sugiere Image Studio
  → create_research_outline → sugiere Chat Paper
    ↓
Banner "EduAI Claw sugiere" aparece en el chat
    ↓
Usuario puede "Ejecutar acción sugerida" → EduAI Claw actúa y muestra resultado
```

**6 salas temáticas:** `#ideas` · `#research` · `#teaching-lab` · `#creative-studio` · `#user-support` · `#anticipation`

**Diseño:** pantalla de inicio donde el usuario elige el tema (no arranca automáticamente), burbujas con avatar y tiempo relativo, Enter para enviar, rooms como pills en el header, input compacto con borrador y pausar integrados.

## 🏫 APl — Planificador Curricular MINEDUC

Agente especializado para docentes chilenos que genera planificaciones rigurosas, completas y alineadas al currículum oficial.

### Cobertura curricular (113 archivos JSON de OA oficiales)

| Nivel | Cursos | Asignaturas por curso |
|-------|--------|----------------------|
| **Parvularia** | Sala Cuna Menor/Mayor · Medio Menor/Mayor · NT1 · NT2 | Ámbitos, Núcleos, OA y OAT por subnivel |
| **Básica 1°–5°** | 1° a 5° Básico | Matemática · Lenguaje · Ciencias Naturales · Historia · Inglés · Tecnología · **Artes Visuales** · **Educación Física y Salud** · **Música** · **Orientación** |
| **Básica 6°–8°** | 6° a 8° Básico | Matemática · Lengua y Literatura · Ciencias Naturales · Historia · Inglés · Tecnología · **Artes Visuales** · **Educación Física y Salud** · **Música** · **Orientación** |
| **Media** | 1° a 4° Medio | Matemática · Lengua y Literatura · Biología · Química · Física · Tecnología · Ciencias para la Ciudadanía · Educación Ciudadana |

### Sugerencias inteligentes para Parvularia (`lib/parvularia-suggestions.ts`)

Sistema de sugerencias automáticas que al escribir el tema del docente detecta los OA, OAT y ámbitos más relevantes:

- **Tokenización y scoring** — analiza el texto con tokenizador sin stopwords y puntúa cada OA/OAT por coincidencias (tokens ≥7 chars = 4 pts, resto = 2 pts)
- **Deduplicación** por id y por ámbito para evitar sugerencias redundantes
- **Output:** `oaSugeridos`, `oatSugeridos`, `ambitosSugeridos` con score y razón para cada sugerencia

### Planificación generada (estructura completa, 8 bloques)

| Bloque | Contenido |
|--------|-----------|
| 📌 **Datos generales** | Tabla con nivel, asignatura, horizonte, sesiones, duración y mes |
| 🎯 **Objetivo(s) de Aprendizaje** | Texto oficial completo del OA + subnivel/ámbito/núcleo/OAT en Parvularia |
| 📊 **Indicadores de evaluación** | 4–6 indicadores observables con nivel de logro (básico/intermedio/avanzado) |
| 🏁 **Objetivos de clase** | Uno o dos objetivos concretos y medibles por sesión *(solo Básica y Media)* |
| 📚 **Propósito de aprendizaje** | 2–3 párrafos con sentido pedagógico, relevancia y conexión curricular |
| 🗂️ **Planificación de clase(s)** | Timing explícito (Inicio / Desarrollo / Cierre con minutos), hasta 4 sesiones |
| ✅ **Evaluación** | Tabla: tipo, momento, instrumento, evidencia esperada y criterios de logro |
| 🌈 **Adaptaciones y diversidad** | Estrategias para distintos ritmos, aventajados y NEE |

### UI del planificador
- **OA seleccionables** — acordeón desplegable con badge contador, etiquetas verde `ámbito · núcleo`
- **OAT / foco transversal** — acordeón independiente con badge contador
- **Sugerencias automáticas de OA** al escribir el tema (Parvularia)
- **Planificaciones guardadas** en Supabase con soporte dual de formato (campos nuevos + compatibilidad legado)

- Botón ← regresa siempre a `/agentes` (no al historial del navegador)
- Límite de tokens: **8.000** (sin cortes de respuesta)

## 🎙️ Audio Lab v2 — Pipeline Robusto con Microservicio Opcional

Sistema rediseñado con arquitectura de pipeline para transcribir y transformar audio de clases, reuniones o cualquier grabación:

```
Audio (MP3/WAV/M4A/MP4/WebM)
    ↓
/api/agents/audio/pipeline
    ├── Modo RÁPIDO → Gemini 2.5 Flash (fallback garantizado)
    └── Modo PRO    → Microservicio externo (Faster-Whisper + WhisperX + pyannote)
                      └── fallback automático a Gemini si falla
    ↓
Guardado en audio_transcriptions (Supabase)
    ↓
/api/agents/audio/export → TXT | MD | SRT | VTT | JSON
    ↓
/api/agents/transcript-editor → 9 operaciones de edición
```

**Modos de transcripción:**

| Modo | Descripción |
|------|-------------|
| ⚡ **Rápido** | Gemini 2.5 Flash — respuesta en segundos, ideal para clases cortas |
| 🎛️ **Pro** | Microservicio externo opcional (Faster-Whisper + WhisperX + diarización pyannote) |

**9 operaciones de edición disponibles:**

| Operación | Descripción |
|-----------|-------------|
| 🪄 **Limpiar** | Corrige puntuación, ortografía y elimina muletillas |
| 📝 **Apuntes** | Convierte una clase grabada en apuntes estructurados |
| 📋 **Acta** | Convierte una reunión en acta formal con asistentes y acuerdos |
| 📄 **Resumen** | Resumen ejecutivo en 5–10 puntos clave |
| ✅ **Tareas** | Extrae tareas y compromisos con responsable y plazo |
| 📚 **Capítulos** | Divide la transcripción en bloques temáticos |
| 🌟 **Highlights** | Extrae frases e ideas más destacadas |
| 🎓 **Guía de estudio** | Genera resumen + preguntas de comprensión + mini quiz |
| ✨ **Personalizar** | Instrucción libre del usuario |

**5 formatos de exportación:** TXT · Markdown · SRT · VTT · JSON

**Características:**
- Archivos hasta **50 MB** (MP3, WAV, M4A, MP4, WebM)
- Detección automática de múltiples hablantes `[Hablante 1]`, `[Hablante 2]`…
- Diarización configurable (UI toggle)
- Timestamps por segmento con timestamps de palabras opcionales
- Historial persistente en Supabase

**Variables de entorno del pipeline:**
```env
AUDIO_PIPELINE_URL=http://localhost:8000
AUDIO_PIPELINE_TOKEN=
AUDIO_PIPELINE_PROVIDER=external
AUDIO_DEFAULT_MODE=quick
```

## 🎨 Image Studio v8 — Multi-proveedor con Rotación de Keys

Motor de generación de imágenes completamente rediseñado con arquitectura modular en `lib/image-config.ts`:

```
Prompt del usuario
    ↓ optimizePrompt() — Gemini 2.5 Flash-Lite (condicional por modo)
    ↓
Cadena de proveedores (configurable por modo vía env vars)
    ↓
Supabase Storage (bucket: generated-images) → URL pública en DB
    ↓
Galería con filtros, fullscreen y descarga
```

**5 proveedores con rotación automática de API keys:**

| Proveedor | Modelos | Keys soportadas |
|-----------|---------|-----------------|
| 🔮 **Gemini Imagen** | `gemini-2.5-flash-image` (GA) · `gemini-3.1-flash-image-preview` · `gemini-2.0-flash-exp` | `GEMINI_API_KEY_IMAGE`, `GEMINI_API_KEY_IMAGE_2` |
| 🌸 **Pollinations** | `flux`, `flux-realism`, `turbo` | `POLLINATIONS_API_KEY` (opcional) |
| ⚡ **Together AI** | `FLUX.2-pro` (aspect_ratio) · `FLUX.2-flex` · `FLUX.1-schnell` · `FLUX.1-dev` | `TOGETHER_API_KEY_1/2/3` |
| 🤗 **Hugging Face** | `FLUX.1-schnell`, `SDXL` | `HF_TOKEN_1..5` |
| 🔑 **OpenRouter** | `gemini-2.5-flash-image` · `gemini-3.1-flash-image-preview` · `riverflow-v2-fast` · `seedream-4.5` · `gpt-5-image-mini` | `OPENROUTER_API_KEY_1/2/3` |

> **Together AI FLUX.2:** usa parámetro `aspect_ratio` en vez de `width/height`. El código detecta automáticamente con `useAspectRatio: true` en la config del modelo.
>
> **Gemini imagen:** el modelo GA oficial es `gemini-2.5-flash-image` (los preview anteriores fueron deprecados en Oct 2025 — quedan como fallback).
>
> **OpenRouter:** cada modelo especifica sus propias `modalities` (`["image","text"]` para Gemini/OpenAI, `["image"]` para Sourceful/ByteDance).

**Orden de proveedores por modo (configurable vía `IMAGE_PROVIDER_ORDER_*`):**

```
fast:         pollinations → openrouter → together → huggingface
quality:      gemini → openrouter → together → huggingface → pollinations
educational:  gemini → openrouter → together → huggingface → pollinations
```

**Optimización de prompt configurable:**

## 📋 Exámenes para Docentes

Sistema completo donde el docente crea pruebas con IA y las comparte mediante un link público:

```
DOCENTE                              ESTUDIANTE
───────                              ──────────
1. Elige tema + distribución
2. IA genera preguntas (sin límite)  ← Groq primario (lotes de 12)
3. Revisa preview con LaTeX          ← ExamMathText en minicard
4. Activa Modo Supervisión (opcional)
5. Publica → obtiene link
6. Comparte por WhatsApp/Classroom    7. Abre link (SIN login)
                                      8. Ingresa nombre + curso + RUT
                                      9. Clic "Iniciar examen" → fullscreen
                                     10. Rinde con timer + seguridad activa
                                     11. IA evalúa desarrollo y V/F
                                     12. Ve nota + retroalimentación + PDF propio
13. Dashboard con 40 alumnos  ←─── resultados en tiempo real
14. Ve incidentes por alumno
15. Análisis pedagógico IA    ←─── /api/exam-report
16. Descarga Excel/PDF (docente)
```

### Cadena de generación IA (sin límite de preguntas)

```
1. Groq — llama-3.3-70b-versatile (PRIMARIO)
   ├── 12 preguntas por lote (seguro bajo 6k TPM free tier)
   ├── 32.768 max_tokens por petición
   └── Fallback entre 4 modelos Groq:
       llama-3.3-70b-versatile → llama-3.1-8b-instant (30k TPM)
       → llama-3.1-70b-versatile → mixtral-8x7b-32768
    ↓
2. OpenRouter (FALLBACK 2)
   meta-llama/llama-3.3-70b-instruct → google/gemini-2.5-flash
   → anthropic/claude-3-haiku → openai/gpt-4o-mini
    ↓
3. Gemini (FALLBACK 3)
   gemini-2.5-flash → gemini-2.0-flash → gemini-1.5-flash
   (503 y 429 tratados igual: skip al siguiente)
```

**Características del examen:**
- **3 tipos de preguntas:** Alternativas (hasta 50), Verdadero/Falso (hasta 30), Desarrollo (hasta 20)
- **LaTeX robusto** — `ExamMathText` con reparación automática de ↑ (U+2191) en preview y en examen online
- **4 niveles de dificultad:** Fácil, Medio, Difícil, Mixto
- **Evaluación IA con Gemini:** desarrollo y justificaciones con puntaje parcial
- **Escala MINEDUC:** notas 1.0–7.0, exigencia configurable (50%–70%)
- **Timer:** 15–120 minutos con alertas visuales
- **Papelera de exámenes:** soft delete con `deleted_at`, restaurar y eliminar permanente
- **Dashboard docente:** tabla, distribución de notas, estadísticas, auto-refresh 15s, columna de incidentes

#### Análisis pedagógico automático (`/api/exam-report`)

API con Gemini 2.5 Flash que genera:
- Estadísticas: promedio, tasa de aprobación, máximo y mínimo
- Las **3 preguntas con menor rendimiento** y las **3 mejor dominadas**
- Distribución de notas por rangos
- **3 párrafos de análisis pedagógico:** análisis general, contenidos críticos, recomendaciones

## 🔒 Sistema de Supervisión de Exámenes (Anti-trampa)

Sistema de cuatro niveles para detectar y registrar conductas sospechosas:

#### Nivel 1 — Protecciones de frontend
- **Fullscreen obligatorio** activado silenciosamente al clic "Iniciar examen"
- **Bloqueo de teclas:** Ctrl+C/V/X/A/P/S/U/R/N, Ctrl+Shift+I/J/C, F12, F11, F5, Alt+F4, PrintScreen
- **Bloqueo de clipboard:** copy, cut, paste
- **`user-select: none`** en todo el contenido del examen
- Detección de blur, visibilitychange y fullscreen_exit

#### Nivel 2 — Política de sanciones progresivas (`lib/exam-guard.ts`)

```
1° incidente → Advertencia: overlay con mensaje, sin bloqueo
2° incidente → Bloqueo 15 segundos con cuenta regresiva
3° incidente → Bloqueo 30 segundos con cuenta regresiva
4° + incidente → Bloqueo 60 segundos + bandera roja (security_flagged)
```

#### Nivel 3 — Auditoría completa (`exam_incidents`)

Cada evento registra: tipo, detalle, pregunta en curso, tiempo restante, estado fullscreen, dimensiones de ventana, user-agent y timestamp.

#### Nivel 4 — Reporte en panel docente

Columna **"Incidentes"** con semáforo de riesgo (0 = limpio · 1–2 = 🟡 · 3–4 = 🟠 · 5+ = 🔴).
