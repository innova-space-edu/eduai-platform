## ВҝQuГ© es EduAI Platform?

EduAI Platform es una plataforma educativa de siguiente generaciГіn que combina **25+ agentes de inteligencia artificial especializados**, un **Creator Hub con 8 formatos de salida potenciados por Gemini 2.5 Flash**, un **Audio Lab para transcripciГіn y ediciГіn de audio con IA**, un **Workspace de proyectos** para organizar todo tu material, un **sistema de exГЎmenes para docentes con evaluaciГіn IA y modo de supervisiГіn antifraude**, y un **panel de administraciГіn completo** para gestionar la plataforma.

Va mГЎs allГЎ de un simple chatbot вҖ” es un ecosistema inteligente donde mГәltiples agentes colaboran en paralelo para que el estudiante aprenda mГЎs rГЎpido y de forma mГЎs profunda. DiseГұada especialmente para el contexto educativo chileno, incluye cobertura curricular completa del **MINEDUC** (Parvularia, BГЎsica y Media) con escala de notas 1.0вҖ“7.0.

### рҹ”’ Sistema de SupervisiГіn de ExГЎmenes (Anti-trampa)

Sistema de cuatro niveles para detectar y registrar conductas sospechosas durante exГЎmenes:

#### Nivel 1 вҖ” Protecciones de frontend

Activadas automГЎticamente cuando el docente habilita **Modo de supervisiГіn** al crear el examen. El estudiante hace clic en "Iniciar examen" y el modo seguro se activa silenciosamente sin avisos previos:

- **Fullscreen obligatorio** activado por el gesto de clic del estudiante
- **Bloqueo de teclas:** Ctrl+C/V/X/A/P/S/U/R/N, Ctrl+Shift+I/J/C, F12, F11, F5, Alt+F4, Escape, PrintScreen, Meta
- **Bloqueo de clipboard:** copy, cut, paste
- **Bloqueo de menГә contextual** y drag
- **`user-select: none`** en todo el contenido del examen
- DetecciГіn de blur, visibilitychange y fullscreen_exit

#### Nivel 2 вҖ” PolГӯtica de sanciones progresivas (`lib/exam-guard.ts`)

```
1В° incidente вҶ’ Advertencia: overlay con mensaje, sin bloqueo
2В° incidente вҶ’ Bloqueo 15 segundos con cuenta regresiva
3В° incidente вҶ’ Bloqueo 30 segundos con cuenta regresiva
4В° + incidente вҶ’ Bloqueo 60 segundos + bandera roja (security_flagged)
```

El `SecurityOverlay` muestra tipo de infracciГіn, nГәmero de incidente, si el examen estГЎ marcado para revisiГіn, y la cuenta regresiva. Los botones de navegaciГіn se deshabilitan mientras dura el bloqueo. El tiempo del examen sigue corriendo.

#### Nivel 3 вҖ” AuditorГӯa completa (`exam_incidents`)

Cada evento registra: tipo, detalle (tecla exacta), pregunta en curso, tiempo restante, estado fullscreen, dimensiones de ventana, user-agent y timestamp. Los datos se envГӯan fire-and-forget desde `ExamGuard` вҖ” nunca bloquean el examen.

**Tipos de evento registrados:**

| Evento | Severidad |
|--------|-----------|
| `tab_hidden` | Alta |
| `copy_attempt` / `paste_attempt` / `cut_attempt` | Alta |
| `print_attempt` / `reload_attempt` | Alta |
| `fullscreen_exit` | Media |
| `contextmenu_attempt` / `blocked_shortcut` | Media |
| `window_blur` / `drag_attempt` | Baja |

#### Nivel 4 вҖ” Reporte en panel docente

Columna **"Incidentes"** en la tabla de resultados con semГЎforo de riesgo:

| Conteo | Nivel | Color |
|--------|-------|-------|
| 0 | Limpio | вҖ” |
| 1вҖ“2 | рҹҹЎ Leve | Amarillo |
| 3вҖ“4 | рҹҹ  Medio | Naranja |
| 5+ | рҹ”ҙ Alto | Rojo |

Clic en el badge abre un **modal de detalle** con cada evento, hora exacta, nГәmero de pregunta y tiempo restante al momento del incidente.

## InstalaciГіn y desarrollo

### Requisitos previos
- Node.js 18+
- Cuenta en [Supabase](https://supabase.com) (gratuita)

### 1. Clonar e instalar
```bash
git clone https://github.com/innova-space-edu/eduai-platform.git
cd eduai-platform
npm install
```

### 2. Variables de entorno
```bash
cp .env.example .env.local
```

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJxxx...
SUPABASE_SERVICE_ROLE_KEY=eyJxxx...   вҶҗ requerido para el panel admin y exam-security

# IA вҖ” Gemini (requerido)
GEMINI_API_KEY=AIzaXxx...

# IA вҖ” Groq (requerido: streaming del tutor + fallback exam-generate)
GROQ_API_KEY=gsk_xxx...

# IA вҖ” OpenRouter (fallback general, opcional)
OPENROUTER_API_KEY=sk-or-xxx...

# ImГЎgenes вҖ” provider primario (opcional)
TOGETHER_API_KEY=xxx...

# ImГЎgenes вҖ” HuggingFace fallback adicional (opcional)
HF_TOKEN_1=hf_xxx...

# Cache y rate limiting вҖ” Upstash Redis (opcional)
UPSTASH_REDIS_REST_URL=https://...upstash.io
UPSTASH_REDIS_REST_TOKEN=...
```

> **Nota вҖ” SUPABASE_SERVICE_ROLE_KEY:** requerida para que `/api/exam-security/event` y `/api/admin` puedan escribir en tablas protegidas. Sin esta key, el registro de incidentes y el panel admin no funcionarГЎn.

> **Nota вҖ” voces del podcast:** Los hosts siempre se llaman **ГҒlvaro** (`es-ES-AlvaroNeural`) y **Elvira** (`es-ES-ElviraNeural`). Los nombres estГЎn fijos en el cГіdigo del renderer.

> **Nota sobre imГЎgenes:** Pollinations AI funciona **sin API key**. Si no configuras `TOGETHER_API_KEY`, la generaciГіn sigue funcionando con FLUX gratuito.

### 3. Supabase вҖ” Migraciones y Buckets

Ejecutar todas las migraciones SQL en **Supabase вҶ’ SQL Editor** en el orden indicado en la secciГіn Base de Datos.

Crear los buckets de Storage:
```
papers          вҶ’ Privado + RLS por usuario
creations       вҶ’ PГәblico
chat-files      вҶ’ PГәblico
workspace-files вҶ’ Privado + RLS por usuario
```

Agregar admins iniciales en la tabla `admin_emails`:
```sql
INSERT INTO admin_emails (email) VALUES ('tu@correo.cl');
```

### 4. Desarrollo
```bash
npm run dev
```
